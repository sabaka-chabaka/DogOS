# DogOS — Roadmap

Дорожная карта разработки DogOS: от зелёного экрана до сети и shell. Пункты идут примерно в порядке зависимости друг от друга — каждый следующий этап опирается на предыдущий.

## Этап 1. Фундамент вывода

- [x] **1.** Загрузка ядра через `bootloader` (UEFI), получение `BootInfo`
- [x] **2.** `FramebufferWriter`: запись пикселя с учётом stride и формата (Rgb/Bgr/U8)
- [x] **3.** Базовые цвета и примитивы: `fill`, `fill_rect`, линии, рамки
- [x] **4.** Растровый шрифт (например `noto-sans-mono-bitmap`) и функция отрисовки символа
- [x] **5.** Текстовая консоль над framebuffer: строки, перенос, прокрутка
- [x] **6.** Управляющие символы: `\n`, `\t`, очистка экрана
- [x] **7.** Макросы `println!`/`print!` поверх консоли (статический `Writer` + `spin::Mutex`)
- [x] **8.** Драйвер последовательного порта (COM1, крейт `uart_16550`) для логов
- [x] **9.** Фасад `log` с бэкендом на serial + экран
- [x] **10.** Информативный panic-хендлер: сообщение и место паники на экран/serial вместо тихого `loop {}`

## Этап 2. Низкоуровневая инициализация CPU

- [ ] **11.** GDT (Global Descriptor Table) через крейт `x86_64`
- [ ] **12.** TSS с отдельным стеком для double fault
- [ ] **13.** IDT (Interrupt Descriptor Table), базовая структура
- [ ] **14.** Обработчик breakpoint exception (`#BP`) для отладки
- [ ] **15.** Обработчик double fault (`#DF`) на отдельном стеке
- [ ] **16.** Обработчик general protection fault (`#GP`)
- [ ] **17.** Обработчик page fault (`#PF`) с выводом адреса и причины
- [ ] **18.** Обработчик invalid opcode (`#UD`)
- [ ] **19.** Ремап PIC (8259) на свободные векторы прерываний
- [ ] **20.** Включение прерываний (`sti`) и базовый диспетчер IRQ
- [ ] **21.** Переход на APIC (Local APIC + IOAPIC) вместо PIC
- [ ] **22.** Обработка spurious interrupt и логирование непредвиденных IRQ

## Этап 3. Таймер и время

- [ ] **23.** Драйвер PIT (Programmable Interval Timer), IRQ0
- [ ] **24.** Счётчик тиков / аптайм ядра
- [ ] **25.** Busy-wait sleep на основе таймера
- [ ] **26.** Чтение времени из RTC (CMOS)
- [ ] **27.** Переход на APIC timer (точнее и без PIC)
- [ ] **28.** Калибровка APIC timer через PIT
- [ ] **29.** Простой таймаут/watchdog API
- [ ] **30.** Монотонные системные часы для будущего планировщика
